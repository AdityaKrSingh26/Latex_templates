\chapter{GRAPH THEORY BACKGROUND}

\section{Graph Theory Fundamentals}
The definitions and algorithmic foundations in this chapter follow the standard treatment in Cormen et~al.~\cite{cormen2022introduction}.

\subsection{Basic Definitions}

\begin{definition}[Graph]
A graph $G$ is an ordered pair $G = (V, E)$ where:
\begin{itemize}
    \item $V$ is a set of vertices (also called nodes),
    \item $E$ is a set of edges, where each edge connects two vertices.
\end{itemize}
\end{definition}

\begin{definition}[Directed Graph]
A directed graph (or digraph) is a graph where each edge has a direction. An edge from vertex $u$ to vertex $v$ is denoted as $(u, v)$ or $u \to v$, and is distinct from $(v, u)$.
\end{definition}

\begin{definition}[Weighted Graph]
A weighted graph assigns a numerical value (weight) to each edge. Formally, it includes a weight function $w : E \to \mathbb{R}$ that maps edges to real numbers.
\end{definition}

\begin{definition}[Adjacency]
Two vertices $u$ and $v$ are adjacent if there exists an edge connecting them. In a directed graph, $u$ is adjacent to $v$ if edge $(u, v)$ exists.
\end{definition}

\begin{definition}[Degree]
The degree of a vertex $v$, denoted $\deg(v)$, is the number of edges incident to $v$. In directed graphs:
\begin{itemize}
    \item \textbf{In-degree:} Number of edges coming into $v$
    \item \textbf{Out-degree:} Number of edges going out from $v$
\end{itemize}
\end{definition}

\subsection{Paths and Connectivity}

\begin{definition}[Path]
A path in a graph is a sequence of vertices $(v_1, v_2, \ldots, v_k)$ such that each consecutive pair is connected by an edge. The length of the path is $k-1$ (number of edges).
\end{definition}

\begin{definition}[Connected Graph]
An undirected graph is connected if there exists a path between every pair of vertices.
\end{definition}

\begin{definition}[Weakly Connected]
A directed graph is weakly connected if replacing all directed edges with undirected edges results in a connected graph.
\end{definition}

\begin{definition}[Strongly Connected]
A directed graph is strongly connected if there exists a directed path from every vertex to every other vertex.
\end{definition}

\begin{definition}[Connected Component]
A connected component of a graph is a maximal subgraph in which every pair of vertices is connected by a path. A graph can have multiple connected components.
\end{definition}

\subsection{Special Graph Types}

\begin{definition}[Directed Acyclic Graph (DAG)]
A directed acyclic graph is a directed graph with no directed cycles. That is, there is no sequence of edges that starts and ends at the same vertex.
\end{definition}

\begin{definition}[Bipartite Graph]
A bipartite graph is a graph whose vertices can be partitioned into two disjoint sets $U$ and $V$ such that every edge connects a vertex in $U$ to a vertex in $V$. No edge connects vertices within the same set.
\end{definition}

\begin{definition}[Complete Graph]
A complete graph $K_n$ is a graph with $n$ vertices where every pair of vertices is connected by an edge.
\end{definition}

\begin{definition}[Tree]
A tree is a connected acyclic graph. A tree with $n$ vertices has exactly $n-1$ edges.
\end{definition}

\subsection{Graph Representations}

\textbf{Adjacency Matrix.} For a graph with $n$ vertices, the adjacency matrix $A$ is an $n \times n$ matrix where:
\begin{itemize}
    \item $A[i][j] = 1$ if edge $(i, j)$ exists,
    \item $A[i][j] = 0$ otherwise.
\end{itemize}
For weighted graphs, $A[i][j]$ contains the edge weight instead of 1.

\noindent Space complexity: $O(n^2)$. Edge lookup: $O(1)$. Finding all neighbors: $O(n)$.

\textbf{Adjacency List.} Each vertex maintains a list of its adjacent vertices.

\noindent Space complexity: $O(n + m)$ where $m$ is the number of edges. Edge lookup: $O(\text{degree})$. Finding all neighbors: $O(\text{degree})$.

\textbf{Implicit Representation.} For some graphs, edges are not stored explicitly but computed on demand using a predicate function. This is memory-efficient when the graph is dense or the edge predicate is simple. Our solution uses implicit representation where edges are computed via the reachability predicate.

\section{Spatial Data Concepts}

\subsection{Geographic Coordinate Systems}

\textbf{Latitude and Longitude.}
\begin{itemize}
    \item \textbf{Latitude} ($\phi$): Angular distance north or south of the equator, ranging from $-90$ to $+90$ degrees.
    \item \textbf{Longitude} ($\lambda$): Angular distance east or west of the Prime Meridian, ranging from $-180$ to $+180$ degrees.
\end{itemize}

\textbf{Coordinate Conventions.}
\begin{itemize}
    \item Standard: (latitude, longitude), used by most mapping applications.
    \item GeoJSON: (longitude, latitude), used in GeoJSON format and some APIs.
\end{itemize}
This inconsistency is a root cause of coordinate swap errors.

\subsection{Distance Calculations}

The central question in our anomaly detection framework is deceptively simple: \textit{could a truck have physically travelled from GPS ping~$p_1$ to the next ping~$p_2$ in the time that elapsed between them?} Answering this requires computing the geographic distance between two coordinate pairs reported by the tracking device. Because the Earth is approximately spherical, the shortest path between two surface points is not a straight line through the interior but an arc along the surface, known as a \textit{great circle}. The Haversine formula computes the length of this arc.

\textbf{Analogy.} Imagine stretching a taut string between two pins on a globe: the string follows the surface curvature, and its length is the great-circle distance, the minimum distance a vehicle must cover between the two locations. The Haversine formula computes this arc length.

\textbf{Haversine Formula.} Given two GPS positions $p_1 = (\phi_1, \lambda_1)$ and $p_2 = (\phi_2, \lambda_2)$ in radians, the great-circle distance is computed as follows. Let $R = 6{,}371$\,km denote the mean Earth radius:
\[
\Delta\phi = \phi_2 - \phi_1, \qquad \Delta\lambda = \lambda_2 - \lambda_1
\]
\[
a = \sin^2\!\left(\frac{\Delta\phi}{2}\right) + \cos(\phi_1) \cdot \cos(\phi_2) \cdot \sin^2\!\left(\frac{\Delta\lambda}{2}\right)
\]
\[
c = 2 \cdot \arcsin\!\left(\sqrt{a}\right), \qquad d = R \cdot c
\]

Here $\Delta\phi$ and $\Delta\lambda$ capture the north--south and east--west angular separations. The term $a$ combines these while correcting for meridian convergence near the poles via $\cos(\phi)$: a 1° longitude difference spans $\approx$\,111\,km at the equator but only $\approx$\,28\,km near the Arctic. The arc $c$ converts the angular separation to a fraction of the Earth's circumference, yielding surface distance $d$.

\textbf{Worked example.} A shipment reports two consecutive pings:
\begin{itemize}
    \item $p_1$: Mumbai $(19.076{^\circ}\text{N},\; 72.878{^\circ}\text{E})$ at 10:00\,AM
    \item $p_2$: Pune $(18.520{^\circ}\text{N},\; 73.856{^\circ}\text{E})$ at 12:30\,PM
\end{itemize}
Applying the Haversine formula yields $d \approx 149$\,km. The elapsed time is 2.5\,hours, giving an implied speed of $149 / 2.5 = 59.6$\,km/h, well below our 250\,km/h threshold. The reachability predicate returns \texttt{TRUE}: this pair of pings is consistent with a single truck travelling on the Mumbai--Pune Expressway.

Now consider a conflicting ping from a second device: $p_2'$: Chennai $(13.083{^\circ}\text{N},\; 80.270{^\circ}\text{E})$ at 10:15\,AM. The Haversine distance from Mumbai to Chennai is $\approx$\,1{,}028\,km, and the elapsed time is just 15~minutes (0.25\,h). The implied speed is $1{,}028 / 0.25 = 4{,}112$\,km/h, clearly impossible for any ground vehicle. The reachability predicate returns \texttt{FALSE}, and this ping becomes a candidate for the multi-vehicle anomaly described in Chapter~1.

\textbf{Accuracy.} The Haversine formula assumes a perfectly spherical Earth. The resulting error is less than 0.3\% compared to ellipsoidal calculations, increasing to approximately 0.55\% for trans-equatorial distances.

\textbf{Vincenty Formula.} Vincenty~\cite{vincenty1975direct} developed a more accurate distance calculation using an ellipsoidal Earth model (WGS-84), achieving accuracy to within 0.5\,mm. It is computationally more expensive due to its iterative nature. For our application, the Haversine formula provides sufficient accuracy with better performance, as positional errors from GPS hardware (3--10\,m under open sky~\cite{kuras2024assessment}) far exceed the sub-metre discrepancy between the two formulae.

\subsection{Speed Constraints and Reachability}
We use 250\,km/h (69.4\,m/s) as a conservative upper bound for ground transport. EU Directive 2002/85/EC mandates speed limiters set to 90\,km/h for heavy goods vehicles~\cite{eurlex2002speedlimiter}, and practical motorway limits across Europe range from 80 to 100\,km/h for trucks. Our threshold of 250\,km/h is deliberately generous to accommodate GPS timestamp inaccuracies, brief high-speed rail segments in intermodal chains, and the need to avoid false positives on legitimate data.

\textbf{Reachability Predicate.} Position $p_2$ is reachable from $p_1$ if:
\[
\text{distance}(p_1, p_2) \leq \text{MAX\_SPEED} \times (p_2.\text{time} - p_1.\text{time})
\]
This forms the basis for edge creation in our reachability graph.
